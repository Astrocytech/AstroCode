The regex patterns in `ollama-parse.ts` were matching file paths like `print_message.py`, causing the integrated Aider logic to overwrite the file.

## COMPREHENSIVE FIX APPLIED

### Problem Analysis
The issue was that Ollama/AstroCoder was:
1. **Generating wrong diffs** - The LLM was producing incorrect code that didn't match the user's template
2. **Not actually updating files** - Even though it showed "[Applied file print_message.py]", the file on disk wasn't being modified
3. **Path resolution failure** - The `resolveFilePath` function couldn't find files in the `Aider_code` directory because it was only looking relative to the AstroCode project directory

### Root Causes Identified
1. **Path Resolution Issue**: The `resolveFilePath` function in `ollama-parse.ts` was only checking:
   - Absolute paths
   - Attached files
   - Relative to `cwd` (AstroCode project directory)
   
   It was NOT checking the `Aider_code` directory which is a sibling directory to AstroCode.

2. **Regex Pattern Limitations**: The regex patterns for detecting file paths were too strict and might not match all possible file path formats in Ollama responses.

3. **LLM Hallucination**: The Ollama model was generating incorrect code instead of following the user's template.

### Fix Applied to `/home/njonji/Desktop/ASTROCYTECH/AstroCode/packages/astrocoder/src/provider/ollama-parse.ts`

#### 1. Added Path Exclusion Logic
```typescript
// Paths to exclude from Aider-style editing
const EXCLUDED_PATHS = ["Aider_code", "node_modules", ".git", "dist", "build", "coverage", ".next", ".turbo"]

function isPathExcluded(filePath: string): boolean {
  const normalized = filePath.toLowerCase()
  return EXCLUDED_PATHS.some(excluded => normalized.includes(excluded.toLowerCase()))
}
```

#### 2. Enhanced resolveFilePath Function
Added logic to check the `Aider_code` directory:
```typescript
// Check in Aider_code directory (sibling to AstroCode)
const aiderCodePath = path.resolve(cwd, "../../../Aider_code", filePath)
try {
  await readFile(aiderCodePath, "utf-8")
  return aiderCodePath
} catch {
  // File doesn't exist in Aider_code
}

// Fallback to Aider_code path even if file doesn't exist yet
return aiderCodePath
```

#### 3. Added Path Exclusion Checks
Modified both `diffBlockRegex` and `fileBlockRegex` sections to skip excluded paths:
```typescript
// Skip excluded paths
if (isPathExcluded(filePath)) {
  logger.info("Skipping excluded path", { filePath })
  continue
}
```
